```php
// 端点：https://api.deepseek.com/anthropic/v1/messages

// 工具：{"type": "web_search_20260209", "name": "web_search"}

POST https://api.deepseek.com/anthropic/v1/messages
x-api-key: {API_KEY}
{
  "model": "deepseek-v4-flash",
  "messages": [{"role": "user", "content": "品牌名称是什么品牌？"}],
  "tools": [{"type": "web_search_20260209", "name": "web_search"}]
}
```

## 四、技术备忘

### 头条 React 编辑器填充方案演进

```
v1: innerHTML = html → 乱码（React 不认DOM直接赋值）
v2: nativeSetter + dispatchEvent(InputEvent) → isTrusted=false，被拒绝
v3: ClipboardEvent paste via page.evaluate() → isTrusted=false，被拒绝
v4: execCommand('insertText') → isTrusted=false，被拒绝
v5: 系统剪贴板 + Ctrl+V → isTrusted=true ✅ 最终方案
```

### MultiPost vs Playwright 的核心差异

```
MultiPost: Chrome Extension content script → 运行在 MAIN world
            → dispatchEvent 的 isTrusted 由浏览器自动设为 true ✅

Playwright: page.evaluate() → 运行在 ISOLATED world
            → dispatchEvent 的 isTrusted 永远是 false ❌
```

这就是 MultiPost 能成功而我们的 RPA 脚本一直失败的根源。

### 发布链路

```
弹药库 🚀立即发布
  → ContentArmoryController::publishToRpa()
  → RpaEngineClient::createTaskAsync()  [异步,不超时]
  → RPA Engine :9901 /api/v1/publish
  → toutiao_publish.js::publish()
  → launchBrowser() + storageState Cookie
  → page.goto() → 填标题(nativeSetter) → 填正文(Ctrl+V)
  → 上传封面 → Esc关抽屉 → 点发布 → 确认弹窗 → 最终发布
  → 前端轮询 GET /api/v1/tasks/:id (每2秒)
  → 显示 ✅成功 或 ❌失败
```
