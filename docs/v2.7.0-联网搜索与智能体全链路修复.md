## 一、联网搜索接入（核心功能）

### 方案对比

| 方式 | 原理 | 适用平台 |
|------|------|----------|
| `tools: [{"type":"search"}]` | 模型调用内置搜索工具 | 豆包(火山方舟) |
| `enable_search: true` | 阿里云千问原生参数 | 通义千问 |
| Anthropic 端点 `web_search_20260209` | DeepSeek 独立搜索端点 | DeepSeek |
| `tools: [web_search]` | 讯飞星火 WebSocket 协议 | 讯飞星火 |
| Playwright RPA 浏览器 | 真实浏览器模拟用户搜索 | 百度AI、元宝 |

### DeepSeek 联网搜索（关键技术点）

DeepSeek 联网搜索不走 OpenAI 兼容端点，需专门调 Anthropic 格式：

```php
// 端点：https://api.deepseek.com/anthropic/v1/messages
// 认证：x-api-key（非 Bearer）
// 工具：{"type": "web_search_20260209", "name": "web_search"}

POST https://api.deepseek.com/anthropic/v1/messages
x-api-key: {API_KEY}
{
  "model": "deepseek-v4-flash",
  "messages": [{"role": "user", "content": "品牌名称是什么品牌？"}],
  "tools": [{"type": "web_search_20260209", "name": "web_search"}]
}
```

### 接入的 8 个 AI 平台

| 平台 | Provider Code | 接入方式 | 工作量 |
|------|--------------|----------|--------|
| DeepSeek | deepseek | OpenAI 兼容 | 已有 |
| 豆包 | doubao | 火山方舟 /v3 + tools:search | 修复 URL + 加搜索 |
| 通义千问 | qwen | DashScope OpenAI 兼容 + enable_search | 修复 provider code |

| Kimi | kimi→siliconflow | 硅基流动免费中转 | 新建 provider |
| 文心一言 | ernie | 千帆 v2 OpenAI 兼容 | 切换adapter+model名 |
| 硅基千问 | siliconflow | 硅基流动免费额度 | 新建 provider+model |
| — | zhipu | GLM-4-9B尝试 | 模型不可用，放弃 |

### 关键经验

1. **API Key 解密必须用 `app(ApiKeyCrypto::class)` 直解析**，不能依赖 DI（Laravel 容器不注入默认值 null 的参数）
2. **火山方舟 endpoint `/v3`** 需特殊处理 URL 拼接
3. **文心一言 v2 用 OpenAI 兼容接口**，不需要 OAuth2 token 交换
4. **Kimi 欠费后用硅基流动免费额度**中转
5. **联网搜索每个平台参数格式不同**，集中在 ScoutAgent 按平台分发

## 四、RPA 引擎优化

### auth-login 端点重写

```javascript
// 统一流程：所有平台（AI + 发布）
chromium.launch() → newContext() → 加载已有Cookie → 等用户关闭浏览器 → storageState() → 验证cookie>2
```

关键修复：
- 移除 `launchPersistentContext`（崩溃根因）
- 加 `--disable-dev-shm-usage` `--disable-gpu`（防闪退）
- Cookie 数量验证（>2 才返回成功）
- 每平台独立 Profile 目录
- 仪表盘加 `no-cache` header

### 仪表盘更新

- 新增「🤖 AI 搜索平台授权」区域（7 平台一键登录）
- 发布平台授权区加缓存状态检查
- 自动默认 workspace 7
- 按钮状态：登录中... → 已登录 ✅
