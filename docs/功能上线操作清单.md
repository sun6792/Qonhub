# Qonhub AI 全渠道一键发布 — 功能上线操作清单

> 基于 v2.1.0 增量升级，所有新增功能已通过全链路测试

---

## 一、环境变量配置

在 `.env` 中追加以下配置（按需）：

```env
# ── 媒介盒子API（新闻媒体发稿） ──
MEDIA_BOX_API_BASE=https://api.meijiehezi.com/v2
MEDIA_BOX_API_KEY=你的媒介盒子API密钥

# ── 头条号OAuth（自媒体发布） ──
TOUTIAO_CLIENT_KEY=你的ClientKey
TOUTIAO_CLIENT_SECRET=你的ClientSecret
TOUTIAO_REDIRECT_URI=https://你的域名/oauth/callback/toutiao

# ── RPA引擎地址（B2B自动化发布） ──
RPA_ENGINE_URL=http://127.0.0.1:9901
```

## 二、需要启动的服务

| 服务 | 命令 | 说明 |
|------|------|------|
| PostgreSQL | 已运行 | 无需变更 |
| Redis | 已运行 | 无需变更 |
| PHP artisan serve | `qonhub.bat 选项3` | 主站，已运行 |
| Queue Worker | `qonhub.bat 选项5` | 需重启以加载新 Job |
| Horizon | `php artisan horizon` | 可选，队列监控仪表盘 |
| RPA Engine | 独立部署 | P2阶段，当前可用API模式 |

## 三、上线执行步骤

### 步骤1：确认数据库迁移

```bash
# 已执行，验证无遗漏
php artisan migrate
# 输出: Nothing to migrate.（已完成）
```

### 步骤2：迁移旧平台授权数据

```bash
# 预览
php artisan migrate:platform-accounts --pretend
# 执行
php artisan migrate:platform-accounts
```

### 步骤3：重启 Queue Worker

```bash
# Windows
taskkill /F /FI "WINDOWTITLE eq Qonhub Worker*"
# 重新运行 watchdog.bat 或 qonhub.bat 选项1

# Docker
docker compose restart queue
```

### 步骤4：清除缓存

```bash
php artisan route:clear
php artisan view:clear
php artisan config:clear
```

### 步骤5：添加测试账号

登录后台 → 全渠道分发运营台（`/geo_admin/content-publish`）→ 添加账号：

```
platform_key: media_box_api
platform_type: news_media
account_name: 媒介盒子主号
credential_type: password
credential_plaintext: 你的媒介盒子API密钥
```

### 步骤6：全链路验证

```bash
# 干跑测试（不实际发HTTP请求）
php artisan test:publish-flow pinshang-sports --dry-run

# 真实测试（需配置API密钥）
php artisan test:publish-flow pinshang-sports
```

### 步骤7：前端验证

| 页面 | 地址 | 验证内容 |
|------|------|---------|
| 运营发布台 | `/geo_admin/content-publish` | 任务列表、创建任务 |
| 任务详情 | `/geo_admin/content-publish/task/1` | 进度条、明细 |
| 客户端发布 | `/client/content-publish` | 列表、新建、详情 |
| 信息锚点 | `/geo_admin/enterprise-anchor/pinshang-sports` | 锚点认证状态 |

## 四、回滚方案

### 数据库回滚

```bash
# 回滚新表（数据会丢失，先备份）
php artisan migrate:rollback --step=3
```

### 路由回滚

恢复 `routes/workspace.php` 到 git 历史版本：
```bash
git checkout HEAD~1 -- routes/workspace.php
php artisan route:clear
```

### 完整回滚

```bash
git stash  # 或 git reset --hard HEAD~1
php artisan migrate:rollback --step=3
php artisan route:clear
php artisan view:clear
php artisan config:clear
```

## 五、已知限制与注意事项

| 限制 | 说明 | 影响 |
|------|------|------|
| 自动发布需API密钥 | 需在对应平台注册开发者账号 | 未配置的平台只能用「手动标记」模式 |
| RPA引擎独立部署 | 无API的平台需额外部署Puppeteer/Playwright服务 | B2B自动化发布暂未开启 |
| 客户端不能直接发布 | 客户提交后由运营审核+执行 | 符合现有代运营模式 |
| 并发限制 | 单账号单日上限+间隔控制 | 大量发布需提前扩容账号池 |
