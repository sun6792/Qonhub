# 豆流 AI — 阿里云生产部署提示词 (for Claude on Alibaba Cloud)

> 复制以下全部内容，发送给阿里云 ECS 上的 Claude Code 执行。  
> 仓库: `https://github.com/sun6792/Qonhub.git` 分支: `feat/v2.2.0-geo-rpa-anchor`

---

## 执行命令

```bash
# ─── 第 1 步：克隆仓库 ───
git clone -b feat/v2.2.0-geo-rpa-anchor https://github.com/sun6792/Qonhub.git /opt/qonhub
cd /opt/qonhub

# ─── 第 2 步：一键 Docker 生产部署 ───
# 自动完成: 环境检测 → Docker安装(如需) → .env.prod配置 → 镜像构建 → 启动全部服务
bash deploy-scripts/geoflow-docker-deploy.sh
```

## 交互式部署时需回答的问题

脚本会依次询问。以下为推荐值（按需修改）：

| 问题 | 推荐值 | 说明 |
|------|--------|------|
| Public web port | `18080` | 对外 Web 端口 |
| Public Reverb port | `18081` | WebSocket 端口 |
| Public APP_URL | `http://你的公网IP:18080` | **必须改成实际 IP 或域名** |
| Admin base path | `geo_admin` | 后台路径前缀 |

## 部署完成后

```bash
# 查看所有容器运行状态
cd /opt/qonhub && docker compose -f docker-compose.prod.yml ps

# 访问地址
# 前台:   http://你的IP:18080
# 后台:   http://你的IP:18080/geo_admin/login
# 默认:   admin / password（首次登录后立即修改！）
```

## 完整容器清单

| 容器 | 端口 | 功能 |
|------|------|------|
| `geoflow-postgres-prod` | 内部 5432 | pgvector 向量数据库 |
| `geoflow-redis-prod` | 内部 6379 | 队列/缓存 |
| `geoflow-app-prod` | 内部 9000 (PHP-FPM) | Laravel 应用 |
| `geoflow-web-prod` | 外部 18080 (Nginx) | Web 服务器 |
| `geoflow-queue-prod` | 内部 | 队列 Worker (7 个队列, timeout=600s) |
| `geoflow-scheduler-prod` | 内部 | 定时任务 (schedule:work) |
| `geoflow-rpa-engine-prod` | 内部 9901 | RPA 浏览器自动化引擎 |
| `geoflow-reverb-prod` | 外部 18081 | WebSocket 实时推送 |

## 环境变量清单（.env.prod 关键项）

部署脚本会自动生成随机密码。如需手动配置，编辑 `/opt/qonhub/.env.prod`：

```ini
APP_ENV=production
APP_DEBUG=false
APP_URL=http://你的IP:18080
APP_TIMEZONE=Asia/Shanghai

DB_CONNECTION=pgsql
DB_HOST=postgres
DB_PORT=5432
DB_DATABASE=geo_flow
DB_USERNAME=geo_user
DB_PASSWORD=<自动生成>

REDIS_HOST=redis
REDIS_PASSWORD=<自动生成>

RPA_ENGINE_API_KEY=<⚠️ 生产环境必须设置为强随机字符串>
RPA_HEADLESS=true

# 以下按需填入
# AI 平台 API Key — 在后台 /geo_admin/ai-models 配置
# 或通过 .env.prod 设置默认值
```

## 安全加固清单（部署后执行）

```bash
# 1. 修改默认管理员密码 → 登录后台 /geo_admin → 用户管理

# 2. 配置 RPA API Key（强随机）
openssl rand -hex 32
# 将输出写入 .env.prod: RPA_ENGINE_API_KEY=<上面生成的随机字符串>

# 3. 配置防火墙（阿里云安全组）
# 开放: 18080 (Web), 18081 (WebSocket)
# 关闭: 5432, 6379, 9901 (仅容器内部访问，不对外)
# 关闭: 22 (仅允许你的 IP)

# 4. 重启使 .env.prod 修改生效
cd /opt/qonhub && docker compose -f docker-compose.prod.yml up -d
```

## 非 Docker 部署（备选方案）

如果阿里云 ECS 内存小于 2GB，Docker 可能跑不动。使用裸机部署：

```bash
bash deploy-scripts/aliyun-setup.sh

# 然后手动启动各服务:
#   主站:     php artisan serve --host=0.0.0.0 --port=18080
#   队列:     php artisan queue:work redis --queue=geoflow,distribution,content-publish,agent_scout,theme-replication,system-updates,default --timeout=600 --tries=3
#   RPA引擎:  cd rpa-engine && npm install && node server.js
#   MCP服务:  cd rpa-engine && npx @playwright/mcp --port 8932

# social-auto-upload 多平台上传（Python 后端）:
#   cd social-auto-upload && pip install -r requirements.txt && python sau_backend.py
```
